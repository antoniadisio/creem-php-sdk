# Security Policy

## Supported Versions

- `1.x` receives security fixes on a best-effort basis as the current stable line.

## Reporting A Vulnerability

- Do not open public GitHub issues for suspected vulnerabilities.
- Use the repository security page and GitHub private vulnerability reporting when it is available: `https://github.com/antoniadisio/creem-php/security`.
- If private reporting is unavailable, contact the maintainer through a non-public contact method listed on the repository owner's GitHub profile and include the affected version, impact, and reproduction steps.
- Initial acknowledgement is targeted within 5 business days.
